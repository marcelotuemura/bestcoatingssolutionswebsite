# Wireframes

Layout references for each route, kept in-repo so design and implementation stay
in sync and are versioned alongside code.

- Organize by route, e.g. `home.md`, `marine.md`, `portal-dashboard.md`.
- Low-fidelity (structure/hierarchy) is enough to start; link high-fidelity
  Figma frames or export PNGs into `docs/assets/` and reference them here.
- Note responsive behavior (mobile-first) and key interactions per section.

Populated in Phase 1–3 as pages are designed (see `ROADMAP.md`). No wireframes
are committed during the foundation phase.
