# `docs/`

Long-form project documentation that complements the top-level strategy docs.

- [`architecture/`](./architecture) — system architecture, decisions (ADRs),
  and the folder-by-folder rationale.
- [`wireframes/`](./wireframes) — low/high-fidelity layout references per route.
- [`branding/`](./branding) — logo, brand assets and usage guidelines.
- [`assets/`](./assets) — source/working files for documentation (diagrams,
  exports) that are not shipped to `public/`.
